# Security Model

NestGate is designed to minimise attack surface.

---

## Core Rules

- Doors never trust phones
- Commands must be signed
- Commands must expire
- Replay is rejected
- Each device has unique credentials

---

## Threats Addressed

- WiFi spoofing
- Replay attacks
- Stolen credentials
- Device impersonation
- Insider misuse

---

## Physical Security

- Controller installed on secure side
- Tamper switch alerts
- Debug ports disabled
- Enclosure locked
